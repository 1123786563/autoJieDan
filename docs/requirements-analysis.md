## 1. 项目概述

### 1.1 项目背景

当前软件外包行业面临**响应效率低、人力成本高、质量不一致**三大痛点。AI 技术的快速发展为自动化接单和交付提供了可能。本项目融合两个开源项目：

| 组件 | 定位 | 技术栈 | 核心能力 |
|------|------|--------|----------|
| **Automaton** | 自主生存型 AI 经济主体 | TypeScript | 经济自主、钱包管理、自我复制、生存约束 |
| **Nanobot** | 超轻量级 AI 代理框架 | Python | 代码生成、多 LLM 接入、多平台消息、技能扩展 |

### 1.2 项目目标

构建一个覆盖**软件外包全生命周期**的自主 AI 代理系统，实现：

1. **商务自动化** — 自动在外包平台发现项目、评估可行性、投标竞标
2. **技术自动化** — 端到端代码生成、测试、部署
3. **运营自动化** — 客户沟通、进度同步、反馈迭代
4. **经济自主** — 自主管理收入、控制成本、维持生存

### 1.3 核心愿景

> **"一个自我维持、自我进化、自我扩展的 AI 服务经济体"**  
> 从"接单执行"到"需求共创"，从"项目交付"到"持续服务"，从"单一代理"到"代理网络"。

## 2. 系统架构需求

### 2.1 双系统分层架构

系统采用 **"治理层 + 执行层"** 的分层设计：

```
┌──────────────────────────────────────────────────────┐
│              治理层 (Automaton 主导)                   │
│  战略目标 → 项目筛选 → 合同风险 → 资源配置 → 经济核算  │
├──────────────────────────────────────────────────────┤
│            ANP 协议通信层 (去中心化)                    │
│  DID 身份 → 消息加密 → 协议协商 → 能力发现             │
├──────────────────────────────────────────────────────┤
│              执行层 (Nanobot 主导)                     │
│  需求分析 → 代码开发 → 测试保证 → 部署运维 → 反馈迭代  │
└──────────────────────────────────────────────────────┘
```

### 2.2 通信协议需求

系统双组件间通过 **ANP (Agent Network Protocol)** 实现去中心化通信：

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| W3C DID 身份 | 每个代理拥有独立 DID 标识（`did:anp:automaton:main` / `did:anp:nanobot:main`） | P0 |
| 端到端加密 | ECDH 密钥协商 + AES-256-GCM 加密通信 | P0 |
| 消息签名 | ECDSA-P256 数字签名，防篡改验证 | P0 |
| 协议协商 | 元协议层支持动态/自然语言协议协商 | P1 |
| 能力发现 | JSON-LD 语义描述，支持代理能力查询 | P1 |
| 双协议兼容 | ANP 协议 + HTTP REST/WebSocket 回退方案 | P0 |

### 2.3 消息类型需求

| 类别 | 消息类型 | 方向 | 说明 |
|------|----------|------|------|
| 任务管理 | `TaskCreate` | Automaton → Nanobot | 创世提示，下发项目任务 |
| 任务管理 | `TaskUpdate` / `TaskComplete` / `TaskFail` | Nanobot → Automaton | 任务状态更新 |
| 状态同步 | `ProgressEvent` | Nanobot → Automaton | 进度报告（百分比、阶段、阻塞项） |
| 状态同步 | `HeartbeatEvent` | 双向 | 心跳检测 |
| 经济相关 | `BudgetUpdate` / `PaymentRequest` | 双向 | 预算更新与支付请求 |
| 协议协商 | `ProtocolNegotiate` / `Accept` / `Reject` | 双向 | 通信协议动态协商 |
| 能力发现 | `CapabilityQuery` / `CapabilityResponse` | 双向 | 代理能力查询 |

### 3.3 双系统协同需求

#### FR-C1: 创世提示（Genesis Prompt）标准接口

Automaton 向 Nanobot 下发任务的标准化接口：

| 字段 | 说明 | 必填 |
|------|------|------|
| `projectId` | 平台来源 + 项目 ID | ✅ |
| `requirementSummary` | 功能描述与技术约束 | ✅ |
| `technicalConstraints` | 必选/禁用技术栈、目标平台 | ✅ |
| `contractTerms` | 总预算、截止日期、里程碑 | ✅ |
| `resourceLimits` | 最大 token 数、最大成本、最大时长 | ✅ |

#### FR-C2: 反馈报告机制

Nanobot 向 Automaton 上报的标准化报告：

| 报告类型 | 内容 | 频率 |
|----------|------|------|
| 进度报告 | 完成百分比、当前阶段、已完成/待完成步骤、预计剩余时间 | 定期（可配置） |
| 资源消耗报告 | token 消耗、API 调用次数、算力成本 | 随进度报告 |
| 异常事件报告 | 阻塞项、错误信息、需要人工介入的决策 | 即时 |
| 质量评估报告 | 测试覆盖率、安全扫描结果、性能指标 | 里程碑完成时 |

## 5. 目标任务类型分析

### 5.1 高匹配度任务（优先承接）

| 任务类型 | 典型场景 | 交付周期 | 自动化潜力 | 综合匹配度 |
|----------|----------|----------|------------|------------|
| **标准化 Web 应用** | 企业官网/CMS、电商平台、SaaS MVP、RESTful API | 1-8周 | ⭐⭐⭐⭐⭐ | **95%** |
| **自动化脚本与工具** | 数据抓取/ETL、RPA、DevOps 工具链、内部工具 | 1-4周 | ⭐⭐⭐⭐⭐ | **92%** |
| **智能合约与 Web3** | DeFi 协议、DAO 工具、NFT 平台、链上数据分析 | 2-6周 | ⭐⭐⭐⭐⭐ | **88%** |

### 5.2 中等匹配度任务（有条件承接）

| 任务类型 | 自动化挑战 | 综合匹配度 |
|----------|------------|------------|
| 移动应用开发 | 原生 API 依赖、设备适配 | 75% |
| 数据工程/ML 管道 | 数据质量依赖、模型调参 | 70% |

### 5.3 低匹配度任务（暂不承接）

- **大规模分布式系统** — 微服务拆分、数据一致性挑战

- **深度领域专业** — 行业 know-how 依赖重
- **高度定制创意设计** — 审美判断、品牌理解

## 6. 风险分析

### 6.1 经济可持续性风险

| 风险 | 触发条件 | 影响 | 缓解策略 |
|------|----------|------|----------|
| 接单成功率波动 | 市场竞争加剧、平台算法调整 | 收入不稳定 | 多平台分散、服务差异化、订阅收入转型 |
| 算力成本上涨 | API 提价、云服务涨价 | 利润率压缩 | 模型降级策略、本地推理(vLLM)、效率优化 |
| 客户付款延迟 | 里程碑争议、客户资金问题 | 应收账款积压 | 智能合约托管、信用评估、保险机制 |
| 加密货币波动 | ETH 暴跌、稳定币脱锚 | 购买力骤降 | 稳定币储备为主、法币通道备用 |

### 6.2 技术交付质量风险

| 风险 | 触发条件 | 影响 | 缓解策略 |
|------|----------|------|----------|
| 需求理解偏差 | 客户表达不清、AI 过度推断 | 交付不符、退款 | 多轮确认、原型验证、里程碑验收 |
| 代码安全漏洞 | 生成代码缺陷、供应链攻击 | 法律责任 | SAST/DAST 扫描、渗透测试、安全编码规范 |
| 性能不达标 | 架构缺陷、负载预估不足 | 合同违约 | 负载测试、性能基准写入合同、自动回滚 |

### 6.3 外包平台集成风险

| 平台 | API 开放度 | 风险 | 缓解策略 |
|------|------------|------|----------|
| Upwork | 高 | 速率限制、数据同步延迟 | 智能缓存、增量同步 |
| Freelancer | 中 | 反爬机制 | 请求频率控制、住宅代理 |
| 猪八戒/码市 | 低 | 反爬对抗、法律风险 | 合规审查、人工兜底 |

| **信任机制** | 智能合约+链上存证 | ✅ 合约保障 | ✅ 争议解决 |
